# TypeScript 프로젝트 감시 모드(다중 파일) 정리

## 1. 프로젝트 초기화

- 여러 TS 파일을 한 번에 관리하려면 **프로젝트 초기화**가 필요

```bash
tsc --init
```

- 명령 실행 시 **tsconfig.json** 생성

- 해당 폴더와 하위 폴더의 모든 TS 파일을 프로젝트 단위로 관리 가능

## 2. 프로젝트 전체 컴파일

- 특정 파일 지정 없이 프로젝트 내 모든 TS 파일 컴파일

```bash
tsc
```

- tsconfig.json이 있는 폴더에서 실행하면 자동으로 모든 `.ts` 파일을 컴파일

- 예: `app.ts`, `analytics.ts` → `app.js`, `analytics.js` 생성

## 3. 감시 모드 적용

파일 변경 시 자동으로 재컴파일

```bash
tsc --watch

# 또는

tsc -w
```

- 프로젝트 내 모든 TS 파일을 감시

- 저장 시 해당 파일만 재컴파일, 변경 사항을 JS 파일에 반영

## 4. 장점

- 여러 파일을 수동으로 지정할 필요 없음

- 변경 즉시 컴파일 및 JS 반영

- 프로젝트 규모가 커져도 tsconfig.json으로 관리 가능
