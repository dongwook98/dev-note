# TypeScript 컴파일러 옵션: target

## 1. 컴파일러 옵션의 목적

- TypeScript 코드를 어떻게 컴파일할지 제어

- 어떤 파일을 컴파일할지, 컴파일되는 코드가 어떤 방식으로 동작할지 정의

- tsconfig.json에서 다양한 옵션을 설정 가능

- 대부분의 옵션은 초기 프로젝트에서는 무시 가능

- 이 코스에서는 중요한 옵션 위주로 학습

## 2. target 옵션

- 역할: TypeScript가 컴파일할 자바스크립트 버전을 지정

- 자바스크립트에 존재하지 않는 기능을 처리할 뿐 아니라, 어떤 브라우저에서 동작할지도 결정

- 초기값: `es5`

  - ES5에서는 `let`과 `const`가 없으므로, 컴파일 시 `var`로 변환됨

  - 구형 브라우저에서도 동작하도록 코드 생성 가능

## 2-1. target 값 예시

- `es3`: 기본값, 구형 브라우저 지원

- `es5`: 모든 코드 호환, 초기 설정

- `es6` 또는 `es2015`: 최신 자바스크립트 기능 지원 (let, const 등)

- 더 최신 버전도 선택 가능

  - 최신 버전을 선택하면 컴파일된 코드가 더 간결하고, 변환할 필요 없는 기능이 줄어듦

## 2-2. target 사용 예시

```json
{
  "compilerOptions": {
    "target": "es6"
  }
}
```

- 위 설정 후 `tsc` 실행 시, ES6 문법 그대로 유지 (`let`, `const`)

- 모던 브라우저용 코드 작성 가능

- 타겟을 높이면 TypeScript가 처리해야 하는 코드 변환 양이 줄어들어 컴파일된 코드가 간결해짐

## 2-3. VSCode 도움 기능

- `ctrl + space`로 자동 완성 기능 활용 가능

- 지원하는 모든 자바스크립트 버전 확인 가능

## 정리:

- `target` 옵션은 TypeScript가 어떤 자바스크립트 버전으로 컴파일할지 결정하는 핵심 옵션

- 구형 브라우저 지원 → 낮은 target

- 모던 브라우저 전용 → 최신 target
