# TypeScript에서 여러 파일 관리 및 tsconfig.json 옵션

## 1. 여러 파일 프로젝트 관리

- 여러 `.ts` 파일을 프로젝트 단위로 관리 가능

- 파일 간 모듈 임포트 가능 (바닐라 JS 모듈과 동일)

- 지금 예제에서는 모든 파일을 `index.html`에서 직접 임포트

- 프로젝트 전체를 관리하면 모든 파일을 한 번에 컴파일 가능

## 2. tsconfig.json 파일

- TypeScript 프로젝트를 관리하는 핵심 파일

- TypeScript에게 파일들을 컴파일하는 방법을 알려줌

- **컴파일러 옵션**을 통해 컴파일러 작동 방식 구성 가능

- 기본적으로 모든 옵션은 주석 처리되어 있어 필요 시 활성화 가능

## 3. exclude 옵션

- 역할: 컴파일에서 제외할 파일/폴더 지정

- 배열 형태로 작성

예시:

```json
{
  "exclude": ["node_modules", "analytics.ts"]
}
```

- 와일드카드 사용 가능:

```json
{
  "exclude": ["**/*.dev.ts"]
}
```

- 의미: 하위 폴더 포함, `.dev.ts`로 끝나는 모든 파일 제외

- `node_modules`는 기본적으로 제외됨 → 별도 설정 불필요

- include와 함께 사용하면 include 대상에서 일부를 제외 가능

## 4. include 옵션

- 역할: 컴파일 대상 파일/폴더 지정

- include에 없는 파일은 컴파일되지 않음

- 예시:

```json
{
  "include": ["src/**/*.ts"]
}
```

- include + exclude 조합 가능

  - include 대상 중 exclude 목록에 있는 파일/폴더는 제외

## 5. files 옵션

- 역할: 개별 파일 단위로 컴파일 지정

- include와 달리 폴더 전체가 아닌 특정 파일만 포함 가능

- 소규모 프로젝트에서 유용

- 예시:

```json
{
  "files": ["app.ts", "analytics.ts"]
}
```

## 6. 기본 권장 사항

- 일반적인 프로젝트:

  - `node_modules` 제외만 신경 쓰면 충분

- 모든 `.ts` 파일을 포함시키고 싶다면 include 사용 필요 없음

- 필요 시 exclude, include, files 옵션을 조합해 컴파일 범위를 세밀하게 제어 가능
