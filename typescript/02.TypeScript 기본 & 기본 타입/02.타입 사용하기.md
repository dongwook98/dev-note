## 1. 자바스크립트의 기본 타입

자바스크립트가 원래 가지고 있는 타입들은 다음과 같다.

- number

- string

- boolean

타입스크립트 역시 이 타입들을 그대로 사용한다.

## 2. 타입스크립트가 제공하는 핵심 장점

### ✔ 타입을 개발 단계에서 강제

- JS는 런타임에서 오류가 터지지만

- TS는 코드를 실행하기 전에 IDE와 컴파일러가 오류를 잡아준다.

### ✔ IDE 지원 강화

- 자동완성, 타입 추론, 실시간 오류 표시 등

- 개발 효율이 크게 상승한다.

### ✔ 최신 JS 기능도 바로 사용

- TS는 최신 자바스크립트 문법을 지원

- 컴파일 시 구형 브라우저에서도 작동 가능한 형태로 변환됨

- Babel과 동일한 역할을 TS 컴파일러가 자체적으로 수행함

### ✔ 타입스크립트 전용 기능 제공

- 인터페이스(Interface)

- 제네릭(Generics)

- 데코레이터(Decorators) (실험적 기능)

- 다양한 타입 유틸리티

## 3. 타입을 명시하지 않으면 생기는 문제

❌ 자바스크립트에서 흔하게 발생하는 버그

```js
function add(n1, n2) {
  return n1 + n2;
}

add('5', 2.8); // "52.8"
```

- 문자열 "5"가 들어가면 숫자 덧셈이 아니라 문자열 결합 발생

- 개발자가 의도하지 않은 결과로 이어짐

## 4. 타입스크립트로 해결하기

✔ 매개변수 타입 지정

```ts
function add(n1: number, n2: number) {
  return n1 + n2;
}

add(5, 2.8); // 정상: 7.8
```

❌ 문자열을 넣으면 컴파일 단계에서 오류 발생

```ts
add('5', 2.8);
```

오류:

```pgsql
Argument of type 'string' is not assignable to parameter of type 'number'.
```

## 5. 타입스크립트 컴파일 과정

- app.ts 작성

- tsc app.ts 실행

- 자바스크립트 파일 app.js 생성

- 브라우저는 **TS가 아니라 JS만 실행**

# 예시 코드 정리

### 1) 타입 없이 작성한 코드 (버그 발생 예제)

```ts
function add(n1, n2) {
  return n1 + n2;
}

const number1 = '5';
const number2 = 2.8;

const result = add(number1, number2);
console.log(result); // "52.8"
```

### 2) 타입스크립트를 사용한 안전한 코드

```ts
function add(n1: number, n2: number) {
  return n1 + n2;
}

const number1 = 5;
const number2 = 2.8;

const result = add(number1, number2);
console.log(result); // 7.8
```

### 3) IDE에서 바로 잡히는 오류 예시

```ts
function add(n1: number, n2: number) {
  return n1 + n2;
}

const number1 = '5'; // ❌ 오류: string → number 불가
```

# 전체 요약

- 타입스크립트는 타입을 통한 안정성 확보와 개발 경험 향상이 핵심

- 런타임 버그를 컴파일 단계에서 차단

- 최신 JS 기능 사용 + 구형 브라우저 대응 가능

- 인터페이스, 제네릭, 데코레이터 등 TS만의 기능도 제공

- IDE와의 조합이 강력해 개발 속도와 품질을 함께 끌어올릴 수 있음
