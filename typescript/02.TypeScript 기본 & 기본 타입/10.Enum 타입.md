## 1. enum이란?

- 타입스크립트에만 있는 **사용자 정의 타입**

- 숫자 또는 문자열에 사람이 읽을 수 있는 **레이블(Label)** 을 붙이는 방식

- 일반 상수 대신 사용하면 **가독성 + 안정성** 향상

```ts
enum Role {
  ADMIN, // 0
  READ_ONLY, // 1
  AUTHOR, // 2
}

const person = {
  name: 'Maximilian',
  age: 30,
  role: Role.ADMIN,
};

if (person.role === Role.ADMIN) {
  console.log('is admin');
}
```

- 기본값: 첫 번째 항목은 0, 이후 항목은 1씩 증가

- 필요 시 숫자 시작값을 바꿀 수 있음

```ts
enum Role {
  ADMIN = 5, // 5
  READ_ONLY, // 6
  AUTHOR, // 7
}
```

- 문자열도 가능

```ts
enum Role {
  ADMIN = 'ADMIN',
  READ_ONLY = 'READ_ONLY',
  AUTHOR = 'AUTHOR',
}
```

## 2. enum 사용의 장점

- 가독성: 숫자나 문자열 대신 레이블로 읽기 쉽게 표현

- 안정성: 잘못된 숫자/문자열 사용 방지

- 재사용성: 코드 어디서든 `Role.ADMIN` 등으로 참조 가능

- 조건문 안전: `if(person.role === Role.READ_ONLY)`처럼 사용 가능

## 3. 자바스크립트 변환

- TS 컴파일 시 enum은 객체 형태로 변환됨

- 각 레이블과 값이 key-value 쌍으로 매핑되어 실제 JS에서도 사용할 수 있음

```js
var Role;
(function (Role) {
  Role[(Role['ADMIN'] = 0)] = 'ADMIN';
  Role[(Role['READ_ONLY'] = 1)] = 'READ_ONLY';
  Role[(Role['AUTHOR'] = 2)] = 'AUTHOR';
})(Role || (Role = {}));
```

> 🔹 내부적으로 숫자 0, 1, 2가 있지만, TS에서는 사람이 읽을 수 있는 레이블로 안전하게 사용 가능

## 4. 언제 enum을 쓰는가?

- **정해진 식별자 목록**을 다룰 때

- 예: 사용자 권한, 주문 상태, 요일, 색상 코드 등

- 단순 상수보다 읽기 쉽고 타입 안정성을 확보할 수 있음
