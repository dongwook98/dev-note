## 타입스크립트에서의 객체(object) 타입

### 1. 객체 타입 기본

- 자바스크립트 객체와 동일한 중괄호 {} 사용

- 키(key)와 값(value) 쌍으로 구성

- 타입스크립트에서는 **객체 타입(object)** 로 처리 가능

- 기본 `object` 타입:

  - "어떤 객체든 상관없다"는 의미

  - 키와 값 타입을 구체적으로 알 수 없음

  - IDE 자동완성이나 타입 검사가 제한됨

```ts
const person: object = {
  name: 'Maximilian',
  age: 30,
};

console.log(person);
```

- 위 코드에서 `person.nickname`과 같이 존재하지 않는 속성 접근 시 TS 오류 발생

- TS는 `person`이 단순 object 타입이라는 것만 알기 때문에, 구체적 속성 정보는 제공되지 않음

### 2. 구체적인 객체 타입 지정

- 객체의 **속성과 타입**을 명시적으로 지정 가능

- 키와 타입을 정의하면, TS가 속성 접근 시 타입 검사 가능

```ts
const person: { name: string; age: number } = {
  name: 'Maximilian',
  age: 30,
};
```

- 이제 TS는 `person.name`은 문자열, `person.age`는 숫자임을 이해

- 존재하지 않는 속성 접근 시 **컴파일 오류 발생**

```ts
person.nickname; // ❌ 오류: 'nickname' 속성이 존재하지 않음
person.age = 'thirty'; // ❌ 오류: number 타입에 string 할당 불가
```

### 3. 타입 추론 활용

- TS는 변수 초기화 값으로 객체 타입을 자동 추론 가능

- 별도로 타입 명시를 하지 않아도 됨

```ts
const person = {
  name: 'Maximilian',
  age: 30,
}; // TS가 { name: string; age: number } 추론
```

- 추론된 타입과 다른 값 할당 시 오류 발생

```ts
person.age = 'thirty'; // ❌ 오류
```

### 4. 요약

- `object` 타입: 모든 객체를 포괄, 구체적 속성 정보 없음

- `{ key: type; ... }`: 객체 구조를 명시, 타입 안전성 확보

- TS는 **타입 추론 기능**으로 초기화 값만 보고도 객체 타입 결정 가능

- TS 컴파일 시 타입 정보는 제거되어 자바스크립트 객체만 남음

- 구체적인 타입 지정은 IDE 자동완성, 타입 검사, 코드 안정성을 크게 향상
