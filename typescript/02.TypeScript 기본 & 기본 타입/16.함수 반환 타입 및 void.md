# 함수 타입 심화 — 반환 타입, void, undefined 정리

함수는 **매개변수 타입**뿐 아니라 **반환 타입(return type)** 또한 타입스크립트에서 중요한 개념입니다.  
이번 내용에서는 반환 타입 추론, 명시적 반환 타입, void, undefined의 차이를 중심으로 정리합니다.

## 1. 타입스크립트의 함수 반환 타입 추론

타입스크립트는 함수의 반환값에 따라 **반환 타입을 자동으로 추론**합니다.

```ts
function add(n1: number, n2: number) {
  return n1 + n2; // number로 자동 추론
}

function addToString(n1: number, n2: number) {
  return (n1 + n2).toString(); // string으로 자동 추론
}
```

- 반환된 표현식의 타입을 분석해 자동으로 판단

- 따로 타입을 명시할 필요가 없음

## 2. 반환 타입을 명시하는 경우

원한다면 명시적으로 타입을 지정할 수도 있습니다.

```ts
function add(n1: number, n2: number): number {
  return n1 + n2;
}
```

하지만 실제 반환 값이 number가 아니라면 오류가 발생합니다.

```ts
function add(n1: number, n2: number): string {
  // ❌ 오류
  return n1 + n2;
}
```

> 일반적으로는 타입 추론을 그대로 사용하는 것이 더 자연스럽고 유지보수에도 좋음

## 3. void 반환 타입 — “아무것도 반환하지 않는 함수”

자바스크립트에는 없는 개념이지만, 타입스크립트에서는 함수가 값을 **반환하지 않을 때** void 타입을 사용합니다.

```ts
function printResult(num: number): void {
  console.log('Result: ' + num);
}
```

특징:

- 반환 구문(`return`)이 없음 → 타입스크립트는 `void`로 판단

- 실제 자바스크립트 런타임에서는 `undefined`가 반환됨

```ts
console.log(printResult(17));
// 출력: Result: 17
// 콘솔 결과: undefined
```

## 4. void vs undefined (타입스크립트 관점)

타입스크립트에서 **void와 undefined는 다르게 취급**됩니다.

### void

- “반환값이 없다”는 의미

- 반환 구문이 없는 함수에서 사용

- 가장 일반적인 패턴

### undefined

- 타입스크립트에서 실제 타입으로 존재

- 변수에 할당 가능

```ts
let someValue: undefined; // 가능
```

하지만 함수의 반환 타입으로는 거의 사용하지 않습니다.

```ts
function test(): undefined {
  // 거의 쓰지 않음
  return; // ❌ 오류 — 실제로 undefined를 반환해야 한다는 의미가 됨
}
```

## 5. 왜 void는 되고 undefined는 안 될까?

- void: “반환 구문이 없다”는 선언

- undefined: “반환 구문은 있지만, 그 결과가 undefined여야 한다”는 의미

즉, 의미 구조 자체가 다름.

## 6. 거의 쓰지 않지만 가능한 코드

```ts
function doNothing(): void {
  return; // 가능
}

function doNothingSpecial(): undefined {
  return undefined; // 가능하지만 일반적으로 사용 안 함
}
```

## 핵심 요약

- 함수의 반환 타입은 타입스크립트가 자동 추론하는 것이 가장 일반적

- 필요할 때만 명시적으로 타입을 지정

- void는 값을 반환하지 않는 함수에 사용 (가장 흔한 패턴)

- undefined는 실제 타입이며, 반환 타입으로는 거의 사용하지 않음

- 런타임에서 void 함수는 기술적으로 undefined를 반환하지만, 타입스크립트는 두 개념을 구별함
