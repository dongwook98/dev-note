# 타입 알리어스(Type Alias)

타입스크립트에서 타입 알리어스(Type Alias)는 기존 타입 또는 유니언 타입을 **새로운 이름으로 재사용**할 수 있게 만드는 기능입니다.

반복적으로 타입을 작성해야 하는 경우, **가독성을 높이고 재사용성을 높이는 방법**입니다.

## 1. 타입 알리어스 정의 방법

```ts
type Combinable = number | string; // 숫자 또는 문자열
type ConversionDescriptor = 'as-number' | 'as-text'; // 특정 문자열만 허용
```

- `type` 키워드로 새로운 타입 이름을 정의

- `=` 뒤에 실제 타입을 할당

- 숫자, 문자열, 불린, 객체, 유니언, 리터럴 타입 등 모든 타입 가능

## 2. 타입 알리어스 사용 예시

```ts
function combine(
  input1: Combinable,
  input2: Combinable,
  resultConversion: ConversionDescriptor
) {
  let result;

  if (resultConversion === 'as-number') {
    result = +input1 + +input2;
  } else {
    result = input1.toString() + input2.toString();
  }

  return result;
}
```

- 함수 매개변수에 직접 타입을 반복 작성하는 대신 타입 알리어스를 사용

- 코드가 간결해지고 의도를 명확하게 전달 가능

## 3. 장점

- 재사용성: 동일한 타입을 여러 위치에서 반복 작성할 필요 없음

- 가독성 향상: 긴 유니언 타입, 리터럴 타입을 단일 이름으로 간결하게 표현

- 오타 방지: IDE가 타입 이름과 허용값을 검사하므로 실수 감소

- 유연성: 숫자, 문자열, 객체, 리터럴 등 어떤 타입이든 알리어스로 정의 가능

## 4. 요약

- 타입 알리어스는 **새로운 이름으로 타입을 정의**하는 방법

- 유니언 타입과 리터럴 타입과 함께 사용하면 **코드 재사용성과 안전성**을 높일 수 있음

- 반복 작성해야 하는 복잡한 타입을 단순화하고, 함수나 변수 선언에서 가독성을 향상
