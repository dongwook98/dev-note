# Any 타입

타입스크립트에서 제공하는 `any` 타입은 가장 유연하지만 동시에 주의가 필요한 타입입니다.

## 1. any 타입의 특징

- **모든 값**을 허용하는 타입입니다.
- 타입스크립트 컴파일러는 해당 변수에 대해 **검사하지 않습니다.**
- 자바스크립트처럼 자유롭게 값을 넣고 수정할 수 있습니다.

```ts
let favoriteActivities: any[] = [];
favoriteActivities.push('스포츠');
favoriteActivities.push(42);
favoriteActivities.push({ name: 'Cooking' });
```

위 코드는 오류 없이 컴파일됩니다.

## 2. 장점

- 유연성: 어떤 값이 들어올지 모르는 상황에서 임시로 사용 가능

- 기존 자바스크립트 코드를 점진적으로 타입스크립트로 마이그레이션할 때 편리

## 3. 단점

- 타입스크립트의 **타입 안정성**을 포기하게 됨

- **런타임 오류 가능성** 증가

- 다른 개발자가 코드를 이해하기 어려움

- `any`를 과도하게 사용하면 사실상 타입스크립트를 쓸 이유가 사라짐

## 4. 사용 권장 방안

- 정말 타입을 알 수 없는 경우에만 임시 사용

- 이후 **런타임에서 검증** 수행

- 가능하면 **구체적인 타입 지정** 또는 `unknown`과 같은 안전한 타입 사용

```ts
let favoriteActivities: unknown[] = [];
```

- `unknown`은 `any`와 비슷하지만, 실제 값을 사용할 때 **타입 체크가 필요**하여 안전성이 높습니다.
