## 1. 튜플(Tuple)이란?

- **고정 길이 배열**이며 각 요소의 타입까지 고정됨

- 일반 배열과 달리, 요소의 순서와 타입이 정확히 정의되어야 함

- 자바스크립트에는 없는 타입스크립트 전용 개념

```ts
const role: [number, string] = [2, 'author'];
```

- 첫 번째 요소: 숫자

- 두 번째 요소: 문자열

- 순서가 바뀌거나 타입이 다르면 컴파일 오류

## 2. 일반 배열과의 차이

- 일반 배열: 길이 자유, 요소 타입 유연

```ts
let roles = [2, 'author']; // 추론: (string | number)[]
roles.push('admin'); // 가능, 길이 제한 없음
roles[1] = 10; // 가능, 타입 유니언(string|number)이므로 오류 없음
```

- 튜플: 길이와 타입 고정

```ts
const roleTuple: [number, string] = [2, 'author'];
roleTuple[1] = 'editor'; // OK
roleTuple[0] = 1; // OK
roleTuple.push('admin'); // TS에서는 허용, 하지만 구조상 주의 필요
roleTuple[2] = 'guest'; // 오류: 튜플 길이 초과
```

> 🔹 주의: `push`는 TS에서 튜플에도 동작하지만, 타입 규칙을 위반하면 런타임 오류 가능.

## 3. 튜플을 사용해야 하는 경우

- 요소 개수와 타입이 명확한 데이터를 다룰 때

- 배열로 처리하면 타입 안전성이 떨어지는 경우

- 예: `[id: number, roleName: string]`, `[lat: number, lng: number]` 등

## 4. 요약

- 튜플: 고정 길이 + 고정 타입

- 일반 배열: 길이 자유 + 요소 타입 유연

- TS에서는 튜플로 데이터를 정확히 정의하면 **타입 안정성과 의도 명시**가 가능
