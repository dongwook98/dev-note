# 함수 타입(Function Types) — 변수를 통해 함수를 안전하게 다루기

함수는 변수에 저장할 수 있고, 그 변수를 통해 호출할 수 있습니다.  
하지만 타입을 제대로 지정하지 않으면 원치 않는 함수가 들어가거나, 타입에 맞지 않는 호출이 일어나 런타임 오류로 이어집니다.  
이를 방지하기 위해 **함수 타입(Function Type)** 이 필요합니다.

## 1. 문제 상황 — `any` 타입의 위험성

```ts
let combineValues: any;
combineValues = add;

console.log(combineValues(8, 8)); // 16
```

- 자바스크립트 자체에서는 문제없음

- 하지만 combineValues가 any라서 어떤 것도 넣을 수 있음

- 예: 숫자를 넣어도 TS는 오류를 찾지 못함

```ts
combineValues = 5; // TS는 허용
combineValues(8, 8); // 런타임 오류 발생
```

## 2. Function 타입 사용 — 하지만 아직 부족함

```ts
let combineValues: Function;
combineValues = add; // 가능
combineValues = printResult; // 이것도 가능 → 원하지 않는 경우
```

### 한계

- Function 타입은 "함수라는 사실만" 보장

- **매개변수 타입**이나 **반환 타입**은 확인하지 못함

- 즉, 잘못된 시그니처의 함수가 들어올 위험이 있음

## 3. 정확한 함수 타입 정의하기 — 화살표 표기법 사용

```ts
let combineValues: (a: number, b: number) => number;
```

의미:

- a, b는 number 타입이어야 함

- 반환 타입도 number이어야 함

이제 올바른 함수만 변수에 저장 가능.

```ts
function add(a: number, b: number) {
  return a + b;
}

function printResult(num: number) {
  console.log('Result: ' + num);
}

combineValues = add; // ✔ OK
combineValues = printResult; // ❌ 오류 발생
```

타입스크립트가 매개변수 개수, 타입, 반환 타입까지 모두 검사합니다.

## 4. 함수 타입을 사용하면 얻는 이점

- 매개변수, 반환 타입이 정확히 맞는 함수만 할당 가능

- 콜백 및 고차 함수에서 특히 유용

- 런타임 오류를 사전에 차단

- 함수의 구조를 명확하게 문서화하는 효과

## 핵심 정리

- 변수를 통해 함수를 다룰 때는 **함수 타입(Function Type)** 으로 정확하게 정의하는 것이 필수

- `Function` 타입은 너무 느슨하므로 지양

- `(a: number, b: number) => number` 형태를 사용해
  매개변수 개수 + 타입, 반환 타입 까지 명확하게 조절

- 잘못된 함수가 들어올 가능성을 타입 단계에서 제거할 수 있음
