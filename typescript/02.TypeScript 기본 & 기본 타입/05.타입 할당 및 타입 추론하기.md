## 타입스크립트에서의 타입 명시와 타입 추론

### 1. 타입 명시

- 함수 매개변수나 변수 선언 시 `:` 뒤에 타입을 지정

- 예시: number, string, boolean

- 자바스크립트에는 없는 문법, 타입스크립트 전용

- 컴파일된 자바스크립트에서는 타입 정보가 제거됨

```ts
function add(n1: number, n2: number, showResult: boolean) {
  // n1, n2는 숫자만, showResult는 boolean만 허용
}
```

- 변수 선언에서도 가능

```ts
const number1: number = 5; // 상수, 숫자 타입
let message: string; // 나중에 문자열 할당 예정
```

### 2. 타입 추론

- 타입스크립트는 변수 초기화 값으로 타입을 자동 추론

- 명시적으로 타입을 지정하지 않아도 TS가 타입을 추론해 오류를 감지

예시:

```ts
const number1 = 5; // TS가 number 타입 추론
let resultPhrase = 'Hello'; // TS가 string 타입 추론
```

- 추론된 타입과 다른 타입을 할당하면 오류 발생

```ts
resultPhrase = 0; // ❌ 오류: string에 number 할당 불가
```

### 3. 타입 명시 vs 추론

- **명시적 타입**: 변수에 어떤 타입이 들어갈지 확실히 지정

- **타입 추론**: 초기화 값만으로 타입을 결정, 중복 작업 최소화

- 일반적으로 **초기값이 있으면 타입 추론만으로 충분**

- 초기화가 없는 변수나 함수 매개변수는 타입 명시 필요

### 4. 요약

- `:` 다음에 오는 타입 키워드는 TS 전용 문법

- 컴파일 시 JS 코드에서는 타입 정보가 제거됨

- 타입스크립트의 핵심 기능: **타입 확인 + 잘못된 사용 시 오류 발생**

- 타입 명시와 추론을 적절히 활용하면 코드 안정성과 가독성 증가
