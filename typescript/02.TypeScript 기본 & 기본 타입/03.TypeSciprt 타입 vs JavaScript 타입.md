## 타입스크립트 vs 자바스크립트: 개발 중 타입 검사

- 자바스크립트는 **동적 타입(dynamic type)**

  - 변수에 처음 숫자를 넣었다가 나중에 문자열을 할당해도 오류 없음

  - 런타임에서 `typeof` 연산자로 타입 확인 가능

- 타입스크립트는 **정적 타입(static type)**

  - 변수와 매개변수의 타입을 **개발 시점에 정의**

  - 타입이 정해지면 런타임에서 변경 불가

  - 잘못된 타입 할당 시 컴파일 단계에서 오류 발생

## 1. 자바스크립트에서 타입 검사 예시

```ts
function add(n1, n2) {
  if (typeof n1 !== 'number' || typeof n2 !== 'number') {
    throw new Error('입력이 잘못되었습니다.');
  }
  return n1 + n2;
}

const number1 = '5'; // 문자열
const number2 = 2.8;

const result = add(number1, number2);
// 런타임에서 오류 발생: "입력이 잘못되었습니다."
```

- `typeof` 연산자로 타입 확인 가능

- 잘못된 값이 들어오면 **런타임 오류** 발생

- 단점: 오류는 **실행 시점**에서만 확인 가능

## 2. 타입스크립트에서 동일 기능

```ts
function add(n1: number, n2: number) {
  return n1 + n2;
}

const number1 = '5'; // ❌ 컴파일 단계에서 오류 발생
const number2 = 2.8;

const result = add(number1, number2);
```

- 매개변수 타입을 지정하면 개발 단계에서 오류를 잡음

- 런타임에서 별도로 검사할 필요 없음

- IDE와 컴파일러가 타입 오류를 바로 알려주므로 버그를 사전에 예방

## 3. 핵심 차이점

- 자바스크립트

  - 동적 타입

  - 런타임에서만 타입 검사 가능

  - 개발 중 실수 발견이 늦음

- 타입스크립트

  - 정적 타입

  - 개발 단계에서 타입 강제

  - 런타임 전에 오류 확인 가능

  - 변수와 매개변수의 타입을 명확히 지정해야 함

- 요약: TS는 개발 중 오류를 사전에 방지하고 코드 안정성을 높이는 도구, JS는 런타임에서 타입을 확인할 수 있는 기본 기능만 제공
