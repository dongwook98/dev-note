## 1. 자바스크립트 vs 타입스크립트의 기본 타입

### ✔ 자바스크립트 기본 타입

- number: 정수/실수 구분 없음

- string: ' ', " ", `` 모두 문자열

- boolean: true / false

### ✔ 타입스크립트가 추가로 제공하는 것

- 더 많은 타입 제공

- 직접 타입 정의 가능

- 유니온 타입

- 타입 별칭

- 인터페이스 등

## 2. JS는 타입을 "알지만", TS는 타입을 "사용한다"

### ✔ 자바스크립트

- 런타임 시점에서 타입을 해석

- 실행해봐야 오류를 알 수 있음

### ✔ 타입스크립트

- 개발 단계에서 타입을 강제

- 실행 이전에 IDE와 컴파일러가 오류를 잡아줌

### ✔ 실행 환경

- 브라우저와 Node.js는 타입스크립트를 직접 실행하지 못함

- 항상 자바스크립트로 컴파일된 결과만 실행됨

## 3. 타입이 없을 때 발생하는 대표적 버그

```ts
add('5', 2.8); // "52.8"
```

"5"는 문자열이므로

- 연산이 문자열 결합으로 동작

의도한 숫자 덧셈이 아닌 "52.8" 발생

JS에서 매우 흔한 논리적 버그

## 4. 타입스크립트로 사전 차단하기

```ts
function add(n1: number, n2: number) {
  return n1 + n2;
}
```

매개변수에 number 타입 지정

잘못된 타입 전달 시 컴파일 단계에서 즉시 오류 발생

## 5. 타입스크립트 컴파일 흐름

- app.ts 작성

- tsc app.ts 실행

- app.js 생성

- 브라우저 혹은 Node.js에서 JS 실행

- 타입 검사는 컴파일 직전 단계에서만 동작한다.

## 전체 요약

- JS는 타입을 실행 중에 판단, TS는 컴파일 단계에서 강제

- 타입 지정만으로 많은 실수를 개발 중에 바로 차단

- 브라우저는 TS를 실행할 수 없으므로 항상 JS로 컴파일됨

- 문자열/숫자 혼합 같은 사소한 버그를 안정적으로 예방

- 타입은 코드 품질, 안정성, 유지보수를 크게 높여줌
