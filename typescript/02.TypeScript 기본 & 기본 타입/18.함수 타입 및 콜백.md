# 콜백과 함수 타입 핵심 정리

## 콜백 함수에 타입을 명시해야 하는 이유

- 함수에 콜백을 전달할 때, **콜백이 어떤 매개변수를 받아야 하는지** 명확하게 해야 한다.

- 타입을 지정하지 않으면 **잘못된 형태의 콜백이 들어와도 컴파일 단계에서 걸러지지 않음**.

## 콜백 타입 정의 방식

```ts
function addAndHandle(a: number, b: number, cb: (num: number) => void) {
  const result = a + b;
  cb(result);
}
```

- `cb: (num: number) => void`

  - 콜백은 number 하나를 받아야 함

  - 반환 타입은 void → 반환값이 있어도 무시한다는 의미

## 함수 타입을 지정하면 얻는 이점

- 콜백이 **올바른 형태로 호출된다는 보장**

- 매개변수 타입이 고정되므로 함수 내부에서 TypeScript가 정확히 추론

- 잘못된 형태의 콜백 전달 시 즉시 오류 표시

## 매개변수는 엄격하게 체크됨

- 콜백이 **매개변수를 두 개 이상** 받으려고 하면 오류

- addAndHandle은 cb에게 **result 하나만 넘기기 때문**

## 반환 타입은 void일 때 느슨하게 취급됨

- 콜백에서 `return result`처럼 값을 반환해도 문제되지 않음

- “반환값을 사용하지 않겠다”고 명시한 것

- void는 “반환값을 사용하지 않는다”는 의미이지
  “반환하지 말라”는 뜻이 아니다.

## 예제 코드

```ts
function addAndHandle(a: number, b: number, cb: (num: number) => void) {
  const result = a + b;
  cb(result);
}

addAndHandle(10, 20, (num) => {
  console.log('결과:', num); // 30
});
```

- `result`는 number라고 정확하게 추론됨

콜백에 `(x: number, y: number) => {}` 처럼 두 개 넣으면 오류

콜백에서 `return 999`를 해도 addAndHandle에서는 무시됨

## 핵심 요약 한 줄

- 콜백의 매개변수 타입·개수는 엄격하게 검사되지만, 반환 타입(void)은 무시되기 때문에 유연하게 동작한다.
