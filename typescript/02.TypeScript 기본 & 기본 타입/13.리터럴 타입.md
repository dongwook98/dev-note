# 리터럴(Literal) 타입

타입스크립트에서 **리터럴 타입**은 특정 변수나 매개변수가 **정확히 어떤 값**을 가져야 하는지를 정의합니다.  
숫자나 문자열과 같은 일반 타입이 아니라, 값 자체를 타입으로 지정하는 방식입니다.

## 1. 리터럴 타입의 특징

- 변수에 단일 값만 허용
- 특정 문자열, 숫자, 불린 등 가능
- 유니언 타입과 함께 사용하면 여러 허용값 지정 가능

예제:

```ts
const number2 = 2.8; // 타입은 2.8
```

- number2는 `number` 타입이 아닌 `2.8`이라는 값 자체를 타입으로 갖습니다.

## 2. 예제: combine 함수에서 리터럴 타입 사용

숫자 또는 문자열을 결합하는 함수에 결과 타입 지정을 위해 사용합니다.

```ts
function combine(
  input1: number | string,
  input2: number | string,
  resultConversion: 'as-number' | 'as-text'
) {
  let result;

  if (resultConversion === 'as-number') {
    result = +input1 + +input2; // 숫자로 변환 후 더하기
  } else {
    result = input1.toString() + input2.toString(); // 문자열로 연결
  }

  return result;
}

const combinedAges = combine(30, 26, 'as-number'); // 56
const combinedStringAges = combine('30', '26', 'as-number'); // 56
const combinedNames = combine('Max', 'Anna', 'as-text'); // "MaxAnna"
```

- `resultConversion`에 'as-number' 또는 'as-text'만 허용

- 다른 문자열을 넣으면 컴파일 에러 발생

- IDE에서 오타를 즉시 확인 가능

## 3. 장점

- 특정 값만 허용하여 **오타나 잘못된 값 방지**

- 유니언 타입과 결합 시 여러 허용값을 명확하게 지정 가능

- 함수의 동작을 **값 단위로 안전하게 제어** 가능

## 4. 유용한 활용 예시

- 상태 관리: `'loading' | 'success' | 'error'`

- 옵션 값 제한: `'small' | 'medium' | 'large'`

- 결과 타입 지정: `'as-number' | 'as-text'`

## 5. 요약

- 리터럴 타입은 값 자체를 타입으로 정의

- 유니언 타입과 결합하면 **허용 가능한 값 목록**을 명확하게 지정 가능

- 오타 방지, 안정적인 코드 작성, 타입 안전성을 높이는 데 유용
