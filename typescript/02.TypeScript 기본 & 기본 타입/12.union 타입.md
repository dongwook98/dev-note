# 유니언(Union) 타입

타입스크립트에서 유니언 타입은 **변수, 매개변수, 상수가 여러 타입 중 하나를 가질 수 있도록 허용**하는 기능입니다.  
숫자와 문자열처럼 서로 다른 타입을 동시에 처리하고자 할 때 유용합니다.

## 1. 예제: 유연한 결합 함수

기존의 `add` 함수는 숫자만 받도록 정의되어 있었습니다.

```ts
function add(a: number, b: number) {
  return a + b;
}

const combinedAges = add(30, 26); // 56
const combinedNames = add('Max', 'Anna'); // 오류 발생!
```

문자열을 넣으면 오류가 발생합니다.

## 2. 유니언 타입으로 매개변수 확장

`number | string`처럼 파이프(`|`) 기호를 사용하여 여러 타입을 받을 수 있습니다.

```ts
function combine(input1: number | string, input2: number | string) {
  let result;

  if (typeof input1 === 'number' && typeof input2 === 'number') {
    result = input1 + input2; // 숫자 연산
  } else {
    result = input1.toString() + input2.toString(); // 문자열 결합
  }

  return result;
}

const combinedAges = combine(30, 26); // 56
const combinedNames = combine('Max', 'Anna'); // "MaxAnna"
```

- `number | string` 타입을 지정하면 숫자와 문자열 모두 전달 가능

- 런타임에서 `typeof`를 사용해 실제 타입을 검사하여 올바른 연산 수행

## 3. 특징 및 주의 사항

- 유연성: 매개변수나 변수에 여러 타입을 허용

- 런타임 검사 필요: 타입이 여러 개일 경우 타입스크립트는 실제 연산 가능 여부를 알 수 없으므로 필요

- 언제 필요한가?

  - 함수가 숫자 또는 문자열을 모두 처리해야 할 때

  - 여러 타입 중 하나가 입력될 수 있는 상황에서

## 4. 요약

- 유니언 타입은 `타입1 | 타입2 | ...` 형태로 정의

- 런타임에서 타입 검사를 통해 안전하게 연산 수행 가능

- 타입스크립트의 타입 안정성을 유지하면서도 **함수를 유연하게 설계**할 수 있음
