## 1) 타입스크립트는 무엇인가?

- **자바스크립트의 슈퍼셋**
  → 완전히 새로운 언어가 아니라, 기존 자바스크립트에 기능을 추가한 언어.
- 자바스크립트 구문을 그대로 사용하면서 **새로운 기능과 장점**을 제공한다.

## 2) 왜 브라우저나 Node.js에서 실행되지 않는가?

- 타입스크립트는 **브라우저·Node가 직접 이해하지 못한다.**
- 따라서 타입스크립트 코드는 **컴파일러가 자바스크립트로 변환**해야 실행할 수 있다.

## 3) 타입스크립트는 언어이자 도구다

- 단순한 언어가 아니라 **TS → JS로 변환하는 강력한 컴파일러**를 포함한다.
- 개발자는 TS 문법과 기능을 사용해도, 최종적으로는 **정상적인 JS 코드가 만들어진다.**

## 4) “JS가 이해하지 못하는 기능을 어떻게 추가할까?”

- 타입스크립트 컴파일러가:
  - 새 기능을 적용된 코드를
  - **복잡한 자바스크립트 코드로 바꿔주는 방식**으로 동작.
- 자바스크립트에서 원래 불가능한 기능을 만드는 것은 아니고,
  **더 간편한 문법·더 명확한 방식**을 제공하는 수준의 기능을 추가한다.

## 5) 타입스크립트의 핵심 기능: 타입 추가

- 런타임(브라우저 실행 중)에 발생할 수 있는 오류를
  **개발 단계에서 미리 감지**할 수 있게 해준다.
- 타입 시스템 덕분에:
  - 논리적 실수
  - 잘못된 데이터로 인한 버그
    등을 작성 중에 바로 확인할 수 있다.

## 6) 타입스크립트가 필요한 이유 — 예제로 설명

### 예시: 두 수를 더하는 함수

자바스크립트에서는 문자열 두 개가 들어오면

`2 + 3 = 5`가 아닌

`"2" + "3" = "23"`

```tsx
// 숫자 2개를 더하는 함수라고 생각하고 작성한 코드
function add(a, b) {
  return a + b;
}

// 하지만 문자열을 넣어도 JS는 에러 없이 실행됨
const result = add('2', '3');

console.log(result); // "23"
```

이라는 **원치 않는 문자열 연결**이 일어난다.

자바스크립트 자체에는 이런 실수를 **사전 감지할 방법이 없다.**

### 해결

타입스크립트는:

- 숫자 아닌 값이 들어오면 컴파일 단계에서 경고를 준다.
- 개발 중 잘못된 호출을 즉시 잡아준다.

→ 결국 **논리 오류를 줄이고 더 안전한 코드 작성이 가능해진다.**
