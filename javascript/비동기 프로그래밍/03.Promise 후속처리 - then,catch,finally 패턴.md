## Promise의 then, catch, finaly

### then

- Promise의 상태가 fulfilled(성공)가 되었을 때 호출됨
- resolve 함수로 전달된 결과값을 인자로 받아 처리함
- then 메서드는 연쇄적으로(chaining) 사용할 수 있으며, 이전 then의 결과를 다음 then에서 전달받아 사용 가능함

### catch

- Promise의 상태가 rejected(실패)가 되었을 때 호출됨
- reject 함수로 전달된 에러를 인자로 받아 처리함
- then으로 연결된 작업 중 어떤 단계에서든 에러가 발생하면 가장 가까운
  catch가 호출됨

### finally

- Promise가 fulfilled(성공)이든 rejected(실패)이든 상태와 무관하게 무조건 실행되는 메서드
- 주로 로딩 상태 표시, 리소스 정리 등의 마무리 작업에 사용됨
- finalㅣy는 인자를 전달받지 않으며, Promise 체인의 마지막 단계에 주로 사용됨

```js
getData()
  .then((data1) => {
    console.log('First:', data1);
    return getData();
  })
  .then((data2) => {
    console.log('Second:', data2);
    return getData();
  })
  .then((data3) => {
    console.log('Third:', data3);
  })
  .catch((error) => {
    console.error('실패:', error);
  })
  .finally(() => {
    console.log('작업 완료 (성공/실패 무관)');
  });
```

> Promise 정의부와 then, catch는 역할이 다르다
> Promise 정의부는 판정 로직을 짜는 자리이고, then과 catch는 그 판정 결과에 따른 대응 로직을 작성하는 자리
