## 동기와 비동기

### 동기 처리

- 코드를 <span style='color:skyBlue'>순차적으로 실행하</span>는 방식
- 앞의 작업이 완료될 때까지 다음 작업은 실행되지 않고 대기함
- 직관적이고 코드 흐름을 이해하기 쉬우나, 오래 걸리는 작업이 전체 프로그램을 차단할 수 있음

```js
console.log('A');

function syncTask() {
  for (let i = 0; i < 1000000000; i++); // 오래 걸리는 작업
  console.log('B');
}

syncTask();
console.log('C');

// 출력 결과:
// A → (작업이 끝난 후) B → C
```

### 비동기 처리

- 특정 코드의 실행이 끝날 때까지 기다리지 않고 <span style='color:skyBlue'>다음 코드를 즉시 실행하는 방식</span>
- 오래 걸리는 작업(네트워크 요청, 타이머 등)을 백그라운드로 넘기고, 그동안 다른 작업이 실행됨
- 코드 흐름이 멈추지 않아 전체 프로그램의 성능이 향상됨

```js
console.log('A');

setTimeout(() => {
  console.log('B');
}, 1000); // 1초 후 실행

console.log('C');

// 출력 결과:
// A → C → (1초 후) B
```

> setTimeout 함수
>
> - 자바스크립트에서 제공하는 타이머 함수
> - 특정 시간이 지난 뒤에, 전달한 콜백 함수를 실행
