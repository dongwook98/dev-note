## 자바스크립트 런타임

- ### 런타임이란?

  - 프로그램이 실제로 실행되는 시간과 공간
  - 코드가 단순히 존재하는 것이 아니라, 돌아가기 위해 필요한 실행 맥락

- ### 런타임 환경

  - 특정 언어가 실행되기 위해 제공되는 실행 도구 모음
  - 자바스크립트의 경우 : 엔진 + API + 이벤트 루프 등으로 구성됨
    - 엔진: 자바스크립트 코드를 해석하고 실행하는 핵심
    - API: 런타임 환경에서 제공하는 기능 집합 (예: Web API, Node.js API)
    - 이벤트 루프: 동기/비동기 작업의 실행 순서를 조율하는 메커니즘
  - 언어만으로는 불완전 -> 반드시 런타임 환경이 필요함

- ### 자바스크립트 런타임 환경 비교
  - #### 브라우저 런타임
    - 자바스크립트 실행을 위한 가장 대표적인 환경
    - 구성: 엔진(V8 등) + Web API(DOM, setTimeout, fetch 등)
    - <span style='color:skyBlue'>화면 조작과 네트워크 요청 등 웹 페이지 중심 기능 제공</span>
  - #### Node.js 런타임
    - 서버 사이드 실행을 위해 만들어진 환경
    - 구성: 엔진(V8) + Node.js API(libuv, 파일 시스템, 네트워크 등)
    - 브라우저 전용 기능(DOM 등)은 없음
    - <span style='color:skyBlue'>백엔드/서버 프로그래밍 중심 기능 제공</span>
