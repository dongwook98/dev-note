## 호이스팅(Hoisting)

- 자바스크립트에서 변수 선언과 함수 선언이 <span style='color:skyBlue'>실행 컨텍스트 생성 단계</span>에서 메모리에 먼저 등록되는 현상
- 실제 코드 위치와 관계없이 선언이 위로 끌어올려진 것처럼 동작함

- ### 특징

  - 변수(var, let, const)와 함수 선언문이 대상
  - 실제 코드 이동이 아닌, 선언 정보만 등록됨
  - 변수 종류, 함수 선언 방식에 따라 초기화 타이밍이 다름

- ### 변수 선언 호이스팅

  - var: 선언 + undefined 초기화 -> 선언 전 참조 <span style='color:skyBlue'>가능</span>(값은 undefined)
  - let, const: 선언만 호이스팅됨, 초기화는 실행 시점에 발생 -> TDZ(Temporal Dead Zone) 동안 참조 <span style='color:lightCoral'>불가</span>

- ### 함수 호이스팅
  - 함수 선언문: 전체 함수가 호이스팅 -> 선언 전 호출 <span style='color:skyBlue'>가능</span>
  - 함수 표현식: 변수에 할당되므로 변수 규칙을 따름 -> 선언 전 호출 <span style='color:lightCoral'>불가</span>

## 예제

```js
// -----변수 호이스팅 ------
console.log(a);
// 실행 전: var a 가 메모리에 등록 + undefined로 초기화
// undefined
var a = 10;

console.log(b);
// 실행 전: let b 가 선언만 등록됨 (초기화 X, TDZ에 존재)
// ReferenceError (TDZ)
let b = 20;

// -----함수 호이스팅------

// 함수 선언문
foo();
// 실행 전: foo 함수 전체가 메모리에 등록됨
// 출력: "함수 선언문"
function foo() {
  console.log('함수 선언문');
}

// 함수 표현식
bar();
// 실행 전: var bar 가 undefined로 초기화됨
// bar는 아직 함수가 아님
// TypeError: bar is not a function
var bar = function () {
  console.log('함수 표현식');
};
```
